## 编译闭环与设计完成条件

当前子任务的实现只有在目标工程编译通过后才算完成。

1. 完成代码修改后，必须从 Build Context 或验证环境中取得当前工程的真实构建命令并执行编译。
2. 如果编译失败，必须读取完整构建日志，定位当前子任务引入的错误，继续修改代码并重新编译。
3. 编译失败时禁止结束当前设计任务，禁止把“已完成代码修改”当作任务完成。
4. 必须持续执行“修改 → 编译 → 读取错误 → 再修改”的闭环，直到构建命令退出码为 0，并且构建日志中没有 error。
5. 不得通过删除原有功能、排除必要源文件或绕过编译目标来获得编译通过。
6. 只有编译通过后，当前设计子任务才具有完成标志，随后才能交给测试子任务执行功能测试和回归测试。
